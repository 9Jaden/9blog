    # 数独

    # 游戏展示：

    # 页面结构绘制：

    使用9*9的二维数组表示数独九宫格

    数独生成步骤【回溯方法】：

    1. 初始化数组，全部fill为0
    2. 对于第一排随机填入1~9的数字【注意用indexOf检查重复与否】
    3. 随后从第二行开始依次遍历数组剩下的位置，随机填入1~9【Math.floor(Math.random() * 9 + 1)】并且满足行、列、小九宫格三重不重复的要求
    4. 如果在一个位置重复填写了十次过后依然无法满足三重不重复条件，则回溯到上一个位置进行重新填充直至遍历完数组所有位置，我们得到一个数独【答案，二维数组表示】

    要点：

    小九宫格重复检查方法---首先确定要检查的位置所在其小九宫格的相对index，用列col%3 + 行rol%3 * 3， 然后以该index为变量进行遍历

        while (index--) {
      // 先减去row%3/col%3为减去九宫格中相对位置到该小九宫格的起始位点，再加index/3或者index%3为加上新index遍历到的位置
      if (
        sudo[row - (row % 3) + Math.floor(index / 3)][
          col - (col % 3) + (index % 3)
        ] === num
      )
        return false;
    }

      const checkLocalNine = (num, curSudo, row, col) => {
    let i = Math.floor(row / 3) * 3;
    let j = Math.floor(col / 3) * 3;
    for (var k = 0; k < 8; k++) {
      if (curSudo[i + Math.floor(k / 3)][j + (k % 3)] == 0) continue;
      if (
        curSudo[i + Math.floor(k / 3)][j + (k % 3)] == num &&
        row != i + Math.floor(k / 3) &&
        col != j + (k % 3)
      )
        return false;
    }
    return true;
  };

    页面使用input标签让用户填充数独

    # 游戏逻辑处理：

    游戏难度选择：初级15中级30高级45，分别对应用户需要填入的数字量

    有两个二维数组，一个是已经生成好的答案数组，另一个是需要重新依照难度等级随机填入空白【数组里重写为0】的现有数组，而后在网页上进行渲染，对于现有数组中为0的对应input框不显示数字，而其他遍历显示数字且readOnly为true【用户不可更改】

    给每一个用户需要填入的input绑定change事件，当input里的value更改后，现有数组curSudo中对应的值也随即更改【实时更新】

    判断游戏是否成功：数独是否完成，遍历现有数组和原已生成数组是否完全一致

    # 独特功能附加：

    数字冲突检测及警告---当用户输入值后进行三重冲突判断，用于实时给用户提供数独数字冲突警告

    用户完成成绩排名---在当前新数独生成且渲染后进行计时，当用户提交正确答案后再次计时，用完成时间进行排名，成绩存储在本地存储localStorage【getItem和setItem】中

    time = ((new Date().getTime() - time) / 1000 / 60).toFixed(2)
得到从开始到结束的时间，以分钟为单位，保留两位小数

    排名：将成绩用含有username和time的对象进行存储，存储进ranks数组，而在本地存储中是用字符串的形式

        if (rankStr) {
      rankStr += `||username=${ranks[i].username}&time=${ranks[i].time}`;
    } else {
      rankStr = `username=${ranks[i].username}&time=${ranks[i].time}`;
    }

    ||区分不同用户，&区分username和time，之后getItem后split获得数据

    而最佳成绩用bestScore额外存储
