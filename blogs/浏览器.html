    浏览器相关

    JIALIANG 2/2/2023 浏览器

    # 引入script时defer和async区别

    script脚本中defer是在文档加载解析后执行，且按顺序

    async则是js加载好之后就会执行 多个async哪个加载好就执行哪个，所以不按顺序

    两者适应场景都主要在于“异步下载”，有多个外部脚本时节省很多时间

    # 浏览器输入URL发生

    1. url解析：判断是搜索内容还是请求URL
    2. 查找本地缓存：如果有缓存直接返回给页面，没有缓存则进入网络请求阶段
    3. DNS解析：域名系统，域名和IP地址相互映射
    4. 通过三次握手建立TCP连接
    5. 合成请求头信息，发送http请求
    6. 处理响应信息
    7. 通过四次挥手断开TCP连接
    8. 如果响应状态码301，则重定向
    9. 浏览器进行页面渲染

    # 页面渲染过程

    • 在解析前，会执行预解析操作，会预先加载 CSS、JS 等文件
    • HTML 解析器解析 HTML，生成 DOM 树
    • CSS 解析器解析 CSS，产生 CSS 规则树
    • 解析 JS 脚本，该过程中需要等待 JS 执行完成才继续解析 HTML

    当浏览器的HTML解析器遇到一个script标记时会暂停构建DOM，然后将控制权移交至JavaScript引擎，这时引擎会开始执行JavaScript脚本，直到执行结束后，浏览器才会从之前中断的地方恢复，然后继续构建DOM，如果对JavaScript的执行位置运用不当，这将会严重影响渲染的速度

    所以会用到defer或async异步下载js脚本

    # 重排与重绘

    重排（尺寸、位置）：当一个元素的位置、尺寸等发生改变的时候 浏览器需要重新计算该元素的几何属性并且摆放到正确的位置的过程就叫做重排。一般像页面初次渲染、带有动画的元素、添加/删除功能、图片放大缩小、浏览器窗口发生改变的时候都会触发重排。

    重绘（外观、样式）：当一个元素的外观、样式发生改变而布局不会改变，重新绘制的过程叫做重绘。

    重排必定会触发重绘，而重绘则不一定会触发回流。一般来说，重排对性能的损耗更多一点

    # router（跳转）和route（参数）

    //$router : 是路由操作对象，只写对象
//$route : 路由信息对象，只读对象

//操作 路由跳转
this.$router.push({
      name:'hello',
      params:{
          name:'word',
          age:'11'
     }
})

//读取 路由参数接收
this.name = this.$route.params.name;
this.age = this.$route.params.age;

    # query和params

    通过url传递参数、控制页面显示数据的两种方式

    query ?xx=yy params (路由path配置/home/:id否则刷新丢失):id

    params：/router1/:id ，    //router1/123，/router1/789 ,这里的id叫做params
query：/router1?id=123 ,  //router1?id=456 ,这里的id叫做query。

    query在路由配置不需要设置参数，而params必须设置，否则刷新后丢失params

    //query:
  {
  path: '/home',
  name: Home,
  component: Home
  }
  
 // params:
  {
  path: '/home/:id',
  name: Home,
  component: Home
  }
