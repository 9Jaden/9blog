    # for in和for of

    遍历数组：for in遍历的是数组的索引（index/key），而for of遍历的是数组元素值（value）

    遍历对象：for in可以遍历对象（获得键）而for of不可以遍历对象（报错）

    for in遍历顺序有可能不是按照实际数组的内部顺序（排序属性：数字属性应该按照索引值⼤⼩升序排列，常规属性：字符串属性根据创建时的顺序升序排列。）

    一个数据结构只要部署了 Symbol.iterator 属性, 就被视为具有 iterator接口, 就可以使用 for of循环

    以下数据结构有

    • 数组 Array
    • Map
    • Set
    • String
    • arguments对象
    • Nodelist对象, 就是获取的dom列表集合

    for of 不同与 forEach, 它可以与 break、continue和return 配合使用,也就是说 for of 循环可以随时退出循环

    # src和href区别

    href标识超文本引用，用在link和a等元素上，href是引用和页面关联，是在当前元素和引用资源之间建立联系

    src表示引用资源，表示替换当前元素，用在img，script，iframe上，src是页面内容不可缺少的一部分。

    src是source的缩写，是指向外部资源的位置，指向的内部会迁入到文档中当前标签所在的位置；在请求src资源时会将其指向的资源下载并应用到当前文档中，例如js脚本，img图片和frame等元素。

    当浏览器解析到src引入js文件的时候会暂停其他资源的下载和处理，直至将该资源加载，编译，执行完毕，图片和框架等元素也是如此，类似于该元素所指向的资源嵌套如当前标签内，这也是为什么要把js放在底部而不是头部。

    当浏览器解析href引入css文件的时候会识别该文档为css文件，会下载并且不会停止对当前文档的处理，这也是为什么建议使用link方式来加载css而不是使用@import。

    补充：link和@import的区别

    两者都是外部引用CSS的方式，但是存在一定的区别：

    区别1：link是XHTML标签，除了加载CSS外，还可以定义RSS等其他事务；@import属于CSS范畴，只能加载CSS。

    区别2：link引用CSS时，在页面载入时同时加载；@import需要页面网页完全载入以后加载。

    区别3：link是XHTML标签，无兼容问题；@import是在CSS2.1提出的，低版本的浏览器不支持。

    区别4：link支持使用Javascript控制DOM去改变样式；而@import不支持。

    # arguments

    在函数调用的时候，浏览器每次都会传递进两个隐式参数：

    一个是函数的上下文对象this，另一个则是封装实参的类数组对象arguments

    存储实际传给函数的参数（实参）的类数组对象，定义是对象！

    类数组，所以具备和数组相同的访问性质及方式，并拥有数组长度属性length，可以通过for(let item of arguments)进行遍历

    转换成真实数组：

    function f5(){
    // 可以使用slice来将arguments转换为真实数组
    var args1 = Array.prototype.slice.call(arguments);
    var args2 = [].slice.call(arguments);
    // 也可以使用Array.from()方法或者扩展运算符来将arguments转换为真实数组
    var args3 = Array.from(arguments);
    var args4 = [...arguments];
}
f5('name', 'age');

    它具有方法：

    arguments.callee 返回正在被执行的function对象，所以可以利用它进行递归

    let sum = function (n) {
    if (n == 1) {
        return 1;
    } else {
        return n + arguments.callee(n - 1); // 5 4 3 2 1
    }
}
console.log(sum(6)); // 21

    使用场景：

    1. 在传入实参不确定时，通过arguments.length查看实参个数与形参个数是否一致
    2. 也可以通过arguments[i]直接更改传入的实参
    3. 遍历参数(for of)
    4. 利用callee实现递归调用
    5. 模拟函数重载（同一个函数完成不同功能）
    // 通过arguments判断传入实参个数完成不同功能
function doAdd() {
    if(arguments.length == 1) {
        console.log(arguments[0] + 5);
    } else if(arguments.length == 2) {
        console.log(arguments[0] + arguments[1]);
    }
}
doAdd(10);  // 15
doAdd(10, 20); // 30

    函数对象的length属性就是函数形参的个数

    function fn (a, b, c) {
    console.log(a, b, c); // 1 2 undefined
    // 函数对象的length属性就是函数形参的个数
    console.log(fn.length); // 3
}
fn(1, 2);

    # js判断变量类型方法

    1. typeof 判断基本类型，判断引用类型时候除了function其他的为object null
    2. instanceof 根据原型链判断，left_proto_ 是否在 right prototype上，遍历左边变量的原型链，直到找到右边变量的 prototype，找到返回true，未找到返回false
    3. constructor 用于引用数据类型，检测方法是获取实例的构造函数判断和某个类是否相同，如果相同就说明该数据是符合那个数据类型的
    4. Object.prototype.toString().call(obj) 所有类型数据都能判断,记住判断结果打印为：'[object Xxx]'

    原理：这句话的意思是让我们用Object原型上的toString方法作用在传入的obj的上下文中（通过call将this指向obj）

    js 中所有对象都是Object 的实例，可以使用 call/apply 调用 Object.prototype 原型对象的 toString() 方法，利用 Object.prototype.toString() 方法可以返回 "[obejct 对象构造函数名]" 的原理来确定数据类型

    # js六种继承方法

    # 1.原型链继-prototype = new建立连线

    function Super(){ this.a=1 }
Super.prototype.say = function(){ console.log(‘hhh’) }
function Sub(){}
Sub.prototype = new Super()

const test = new Sub()

    优点：通过原型继承多个引用类型的属性和方法

    缺点：子类共享引用属性（Sub原型变成了Super的实例，Parent 中的引用属性会被每个子类示例共享，会有污染问题）

    # 2. 构造函数继承-call

    function Super = function(){ this.a = 1 }
function Sub = function(){
       Super.call(this)
       this.b = 2
}
const test = new Sub()

    优点：避免了子类示例共享引用属性的情况

    缺点：如果Parent中存在一个函数，那么每次实例化Children的时候，都会创建一个同样函数，函数的复用性就难以体现；无法访问所继承对象原型上的方法

    # 3. 组合继承（原型链+构造函数）

    function Super(){ this.a=[1,2] }
Super.prototype.say = function(){ console.log(‘hhh’) }
function Sub(){
    Super.call(this)
    this b=2
}
Sub.prototype = new Super()
 
const test1 = new Sub()

    优点：集合了原型继承和构造函数继承的优点，应用最广泛

    避免了子类共享引用属性同时避免了父类构造函数重复对function属性的创建

    缺点：super调用两次效率低

    # 4. 原型式继承-Object.create

    //es5
const obj = { a:1 }
function createObj(o){
    const Fn(){}
    Fn.prototype = o
    return new Fn()
}
const test = createObj(obj)

//es6
const obj = { a:1 }
const test = Object.create(obj)

    优点：对一个对象进行浅克隆创建另一个对象，同时继承该对象的原型属性

    缺点：和原型链继承一样，后代实例会共享父类引用属性（由于是浅克隆，所以实例共享的对象属性如果是引用值，会受被同时被更改）

    const obj = { a:[1,2], b:2 }
const test1 = Object.create(obj)
const test2 = Object.create(obj)

test1.a.push(3)
test1.b=3
console.log(test1.a, test2.a)// [1,2,3]  [1,2,3]
console.log(test1.b, test2.b)// 3 2

    # 5. 寄生式继承：构造函数模式+工厂模式

    function createObj(o){
    let clone = objectCopy(o)
    clone.say=function(){
        console.log(‘hhh’)
    }
    return clone
}

const obj = { a:1 }
const test = createObj(obj)

    优点：根据一个对象克隆创建另一个对象，并增强对象

    缺点：同【盗用构造函数继承】方法在每个实例上都要创建一遍

    和原型链继承一样，parent中的引用属性，会被所有示例共享

    注意：objectCopy不是原生接口，是自定义方法，对入参对象进行复制

    # 6. 寄生式组合继承：构造函数继承 + 原型式继承

    function Super(){ this.a=[1,2] }
Super.prototype.say = function(){ console.log(‘hhh’) }
function Sub(){
    Super.call(this)
    this b=2
}

Sub.prototype = Object.create(Super.prototype)
Sub.prototype.constructor = Sub

const test = new Sub()

    优点：集合了【原型式继承】和【构造函数继承】的优点，效率比【组合继承】更高

    和组合继承一样，只不过没有组合继承的调用两次父类构造函数的缺点

    # map和forEach区别

    map会分配内存空间并存储一个新数组，不会改变原数组，返回新数组

    forEach直接对于数组每个元素执行函数操作，改变原数组，不返回值
