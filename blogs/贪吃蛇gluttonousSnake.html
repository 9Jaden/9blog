    # 贪吃蛇

    # 游戏展示：

    # 页面结构绘制：

    用16*16的数组tableArr表示游戏盘底层数据，而后两次遍历该数组，第一次遍历创建tr第二次遍历创建td

    三要素---头部+身体+食物【利用构造函数创建并渲染】：

    function Head(x, y) {
  this.x = x;
  this.y = y;
  this.render = function (x) {
    tableArr[this.y][this.x].head = x;
  };
}
// 身体和食物render的属性为body和food

    而后依据格子的head/body/food属性渲染不同的样式代表头/身体/食物

    对于身体，以为不止一个，定义bodyArr身体数组

    # 游戏逻辑处理：

    键盘控制【要点：try和catch判别出界】【精髓：身体中间部分不动，最末端格子移动到最前端（原head）】：

    定义up/down/left/right四个方向按键后调用函数

    普通移动【未吃食物】【try，catch是否碰到边界】：

      try {
    // 原来的头不要了
    head0.render(0);
    head0.y -= 1;
    // 更改y后重新渲染新格子
    head0.render(1);
  } catch {
    // 出界，游戏失败
    clearInterval(timer);
    alert("oops!你输啦！");
    location.reload();
  }
  // 调用吃食物函数
  eat();
  // 尾部换到原头部所在位置
  bodyArr[bodyArr.length - 1].render(0);
  bodyArr[bodyArr.length - 1].x = head0.x;
  bodyArr[bodyArr.length - 1].y = head0.y + 1; // 因为原head已经更改
  bodyArr[bodyArr.length - 1].render(1);
  // 数组顺序也要更改
  bodyArr.unshift(bodyArr.pop());
  renderStyle();
  // 调用碰到身体与否的函数
  hitBody();

    除此之外需要记录上一次按的按键，限制转向，比如当蛇在往下走的时候只能左右转不能直接向上回头

    吃到食物后：

    a. food与head重合后food消失，分数+1；

    b. 随机生成food且不能与身体重合；

    c. 生成新body加入bodyArr

        // 生成新body加入bodyArr
    let newBody = new Body(
      // 直接生成放在最后一个原有的位置即可，因为移动和吃同时进行
      bodyArr[bodyArr.length - 1].x,
      bodyArr[bodyArr.length - 1].y
    );
    newBody.render(1);
    bodyArr.push(newBody);

    游戏失败：撞到墙or吃到自己身体 tableArr[head0.y][head0.x].body == 1

    自动移动：定时器setInterval不断调用上一次按键对应的函数

    速度加快：随着分数增加到20/40...定时器事件不断减小150/125/100

    游戏成功：身体长度bodyArr的长度到255时，因为有个头，一共256个格子

    空格键暂停：监听keydown，且e.keyCode==32清除定时器对应timer
