    # 同步请求和异步请求

    同步请求：客户端向服务器发送请求-->等待服务器响应-->处理完毕返回，客户端浏览器没有做别的事情。 同步连接请求数据时，当数据还未请求成功之前，用户界面做任何操作都是无效的，都是不会响应的。只有当请求数据完毕之后，才会响应用户交互，所以通常会卡死主线程。

    异步请求：通过事件触发请求-->服务器处理请求-->处理完毕返回，但是客户端浏览器可以接着做别的事情 当异步请求时，会有单独的子线程去请求数据，而主线程依然响应处理用户交互，所以此时用户交互得到处理，用户流畅操作，用户体验比较好，所以开发过程中最多的还是异步链接

    同步——如果使用者在服务运行的过程中阻塞时崩溃了，当它重新启动时，将无法重新连接到正在进行的调用，所以响应丢失了。使用者必须重复调用过程，并且期望这次不会崩溃。 异步——如果使用者在发送了请求之后等待响应时崩溃了，当它重新启动时，可以继续等待响应，所以响应不会丢失。

    # ajax、fetch、axios三者区别

    分别是术语、api和库，都用于向服务器发送http网络请求

    # Ajax

    ajax是js异步技术的术语，早起相关的api是xhr，它是一个术语：Asynchronous JavaScript And XML，翻译过来就是“异步的 Javascript 和 XML”

    现在衍生成只要用脚本发起通信就可以叫做ajax通信

    XML是文档格式，但现在服务器返回基本为JSON格式数据了

    创建过程：

    1.创建XMLHttpRequest对象,创建一个异步调用对象.

    2.创建一个新的HTTP请求,并指定该HTTP请求的方法、URL及验证信息.

    3.设置响应HTTP请求状态变化的函数.

    4.发送HTTP请求

    <body>
  <script>
    function ajax(url) {
      const xhr = new XMLHttpRequest();
      xhr.open("get", url, false); // true表示请求同步
      xhr.onreadystatechange = function () {
        // 异步回调函数
        if (xhr.readyState === 4) {
            // 4-请求结束
          if (xhr.status === 200) {
            console.info("响应结果", xhr.response)
          }
        }
      }
      xhr.send(null); 
    }
    ajax('https://smallpig.site/api/category/getCategory')
  </script>
</body>

    XMLHttpRequest实例重要方法与事件

    XMLHttpRequest.open(method, url, async)

    ​ 指定HTTP请求方法、目标url、异步与否【boolean】

    XMLHttpRequest.send(null)

    ​ 发起请求，null为所带数据体，post请求需要指定

    事件xhr.onreadystatechange

    ​ 对不同状态进行不同处理，当xhr的readyState为4表示通信成功，可处理返回数据

    一个相对完整的XMLHttpRequest至少需要监听两个事件(onload、onerror)来实现成功和失败的回调，以及调用open()和send()

    # Fetch

    Fetch 是一个 API，它是真实存在的，它是基于 promise 的

    fetch是一种HTTP数据请求的方式，是XMLHttpRequest的一种替代方案【避免回调地狱】，fetch()方法返回一个Promise解析Response来自Request显示状态（成功与否）的方法。使用Promise API更有利于异步代码的书写。

    fetch(url)
  .then(
    function(response) {
      if (response.status !== 200) {
        console.log('Looks like there was a problem. Status Code: ' +
          response.status);
        return;
      }
      // Examine the text in the response
      response.json().then(function(data) {
        console.log(data);
      });
    }
  )
  .catch(function(err) {
    console.log('Fetch Error :-S', err);
  })

    fetch()请求返回的response是Stream对象，因此我们调用response.json时由于异步读取流对象所以返回的是一个Promise对象，需要再then一下

    或者使用async/await优化，减少回调，更加语义化便于理解

    async function geturl(){
	try{
		let res = await fetch(url)
		if(res.status == 200){
			console.log(await res.text()) //同样异步读取
		}
	} catch(err){
		console.log(err)
	}
}

    # Axios

    Axios是一个基于 promise 封装的网络请求库，它是基于 XHR 进行二次封装

    调用请求函数--->请求参数处理--->修改请求头&配置用户标识【请求拦截】--->发起请求--->网络错误处理&授权错误处理&普通错误处理&请求完成【相应拦截】--->返回参数处理

    # 基础请求流程

    一是 请求进入请求拦截前 、二是 真正发起的请求 、三是 请求从响应拦截出来后 。

    这其中可以归为两类， 一类是 针对单独接口的处理
    二类是 针对所有接口需要的内容

    • 针对单独接口的处理

      • 请求前的参数处理
      • 请求后的返回值处理

          res = result.data;

    # 拦截器

    请求拦截器，在发送请求前做一些事情，interceptors给config的headers请求头添加一个字段，即添加用户独立的token（相当于是对每个接口里相同操作的一个封装）

    // 更改请求头
export const handleChangeRequestHeader = (config) => {
  config["xxxx"] = "xxx";
  return config;
};
// 配置身份验证
export const handleConfigureAuth = (config) => {
  config.headers["token"] = localStorage.getItem("token") || "";
  return config;
};
// 请求拦截axios.interceptors.request.use((config)=>{})
axios.interceptors.request.use((config) => {
  config = handleChangeRequestHeader(config);
  config = handleConfigureAuth(config);
  return config;
});

    # Ajax封装

    function ajax(options) {
    // 存储默认值的对象
    var defaults = {
        type: 'get',
        url: '',
        data: {},
        header: {
            'Content-Type': 'application/x-www-form-urlencoded'
        },
        success: function() {},
        error: function() {}
    }

    // options对象覆盖defaults对象中的属性
    Object.assign(defaults, options);

    // 创建ajax对象
    var xhr = new XMLHttpRequest();
    // 拼接请求参数的变量
    var params = '';
    // 循环用户传递进来的对象格式参数
    for (var attr in defaults.data) {
        // 将对象参数转化为字符串格式
        params += attr + '=' + defaults.data[attr] + '&';
    }
    // 将参数最后的&去掉
    params = params.substr(0, params.length - 1);
    // console.log(params);
    // 判断请求方式
    if (defaults.type == 'get') {
        defaults.url = defaults.url + '?' + params;
    }

    // 配置ajax对象
    xhr.open(defaults.type, defaults.url);

    // 发送请求
    if (defaults.type == 'post') {
        // 用户设置的向服务器端传递的请求参数类型
        var contentType = defaults.header['Content-Type'];
        // 设置请求参数格式的类型
        xhr.setRequestHeader('Content-Type', contentType);
        // 判断用户希望的请求参数格式类型
        if (contentType === 'application/json') {
            // 如果类型为json,向服务器端传递json数据格式的参数
            xhr.send(JSON.stringify(defaults.data));
        } else {
            // 向服务器端传递普通类型的参数
            xhr.send(params);
        }
    } else {
        xhr.send();
    }

    // 监听xhr对象下面的onload事件
    // xhr对象接收完响应数据后触发
    xhr.onload = function() {
        // 获取响应头中的数据
        var contentType = xhr.getResponseHeader('Content-Type');
        // 服务器返回的数据
        var responseText = xhr.responseText;
        if (contentType.includes('application/json')) {
            // 将json字符串转换为json对象
            responseText = JSON.parse(responseText);
        }

        // http状态码为200
        if (xhr.status === 200) {
            // 请求成功，调用成功的函数
            defaults.success(responseText, xhr);
        } else {
            // 请求失败，调用失败的函数
            defaults.error(responseText, xhr);
        }
    }
}

    # axios请求函数+promise

    /* ajax请求函数模块
返回值： promise 对象(异步返回的数据是: res.data)*/
import axios from 'axios'
export default function ajax(url, data = {}, type = 'GET') {
return new Promise(funtion(resolve,reject){ //resolve, reject函数类型（高阶函数）
//执行异步请求
    let promise
    if(type === "GET"){
        //准备url query参数数据
        let dataStr =''//数据拼接字符串
        Object.keys(data).forEach(key =>{
            dataStr += key + '=' + data[key] + '&'
        })
        if(dataStr !==''){
            dataStr = dataStr.substrings(0,dataStr.lastIndexOf(&))
            url = url + '?' +dataStr
        }
        //发送GET请求
        promise = axios.get(url)
    }else{
        promise = axios.post(url,data)
    }
    //异常处理
    //成功调佣函数
    promise.then(funtion(response){
        resolve(response.data)
    })
    //失败调用函数
    .catch(funtion(error){
        reject(error)
    })
})
}

    # 同源策略

    URL包括：协议 域名 端口 路径 查询参数

    同源策略要求协议、域名、端口相同，是为了保护本地数据不被JS获取回来的数据污染

    过程：客户端请求发送了--->服务器响应了--->数据回来了--->但是不被浏览器接收

    # 跨域问题

    解决方案：

    CORS(服务端设置ACAO)/nginx反向(中转nginx服务器转发请求)/node正向(服务端请求不被限制，代理服务器)/jsonp(script标签不被限制)/webpack中设置proxy

    CORS：全域资源共享 --- 服务端设置Access-Control-Allow-Origin---需要浏览器和后端同时支持

    //指定允许其他域名访问
‘Access-Control-Allow-Origin:http://172.20.0.206’//一般用法

//是否允许后续请求携带认证信息（cookies）,该值只能是true,否则不返回
‘Access-Control-Allow-Credentials:true’

//允许所有域名都可以访问，不够安全
response.setHeader(“Access-Control-Allow-Origin”,"*");
// 这个标识是服务端JAVA后端设置的

    基于nginx配置请求的CORS：目前很多请求都不是直接暴露的，很多通过nginx做反向代理，因此使用nginx配置固定请求的Access-Control-Allow-Origin，实现跨域访问。方式是在被请求的接口，配置location代理，添加header实现

    nginx反向代理：前端访问相同域名，nginx再根据需要把请求转发到外部域名（类似node中间件代理，搭建中转nginx服务器转发请求）

    node正向代理：利用服务端请求不会跨域的特性，让接口和当前站点同域（代理服务器）

    JSONP：利用script标签没有跨域限制的这个特性来完成的

    webpack中配置proxy来快速获得接口代理的能力

    websocket：双向通信协议（保持前后端实时通信），建立连接之后，sever和client都能主动向对方发送或接受数据
