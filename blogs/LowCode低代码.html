    # LowCode低代码平台

    该项目为基于React技术栈以及React-DnD拖拽库搭建的一款低代码编辑器，可作为一种能够极大提升开发效率的PaaS软件

    # 项目展示

    # 技术栈

    TypeScript[语言] + React[框架] + React-DnD[拖拽库] + pnpm[包管理] + Webpack[打包] + Koa[后端] + pm2[部署]

    # 页面布局

    该编辑器主要由「组件区」、「画布区」、「属性编辑区」这三个部分组成

    组件区负责展示可拖拽的组件(和组件间的层级关系)

    ​ 实现功能：组件可拖拽--->拖拽后与画布区交互 & 组件数据上传后端

    画布区负责渲染拖拽的组件，进行可视化展示

    ​ 实现功能：生成组件相应数据格式--->渲染组件--->选中组件与编辑区交互

    属性编辑区负责编辑所选中组件的相关属性

    ​ 实现功能：处理属性变更--->反馈画布区 & 组件数据更改

    # 重点功能

    # 1. 拖拽组件

    采用React官方实现的拖拽库React-DnD

    在React-DnD中定义了drag和drop两种类型组件

    在组件区中使用useDrag钩子

    定义拖拽组件的type

    export default function ButtonComponent() {
  const [_, drag] = useDrag(() => ({
    type: COMPONENT_TYPE.BUTTON,
  }));

  return (
    <div className="button-component" ref={drag}>
      按钮组件
    </div>
  );
}

    在画布区中使用useDrop钩子

    定义可接受drop组件的type

    利用monitor.getClientOffset()获得拖拽位置

    setData保存组件相关数据

    const [, drop] = useDrop(() => ({
    accept: [
      COMPONENT_TYPE.TEXT,
      COMPONENT_TYPE.BUTTON,
      COMPONENT_TYPE.INPUT,

      const { x, y } = monitor.getClientOffset();

    # 2. 画布渲染

    画布区相对定位，组件绝对定位

    根据底层数据中组件的left和top等属性定位并渲染组件在画布中

    并且根据组件不同的type进行不同类型不同标签的渲染

    # 3. 属性联动

    画布区渲染组件时添加点击事件并传入组件类型和id

    点击组件后，属性编辑区中根据传入的组件id找到组件底层数据，再根据组件类型进行不同类别的编辑区界面渲染

    属性更改完毕后，点击保存按钮将根据编辑区input框内容更新底层数据

    画布区据底层数据更改重新渲染可视化

    # 4. 组件通信

    组件底层数据保存在画布组件中，以便于渲染和更改组件数目

    通过触发根组件函数将组件底层数据传给根组件

    根组件利用props将组件底层数据、组件类型、组件id等在不同子组件中进行通信

    # 5. 后台交互

    将编辑器JSON数据利用axios向后台发起post请求

    后台接收到数据后将其存储到数据库中【koa框架】
