    GMALL电商平台

    JIALIANG 2/2/2023 VuexVue-routerMock.jsAxiosSwiperElement-ui

    # GMALL电商平台

    项目为包括首页、搜索列表、商品详情页、购物车、订单、支付、用户登录/注册等多个子模块的在线电商 SPA，项目采用模块化、组件化、工程化的模式开发

    # 项目展示

    # 技术栈

    Vuex + vue-router + Mock.js+ Axios + Swiper + Element-ui

    # 重点功能

    使用 vue-router 实现页面路由跳转构建单页面应用，并设立导航守卫拦截不合理路由跳转

    通过面包屑词条增删以及点击排序实现交互性高的商品搜索页面

    利用组件通信对非业务逻辑功能的组件（如轮播图、分页器等）进行抽离和封装

    封装 axios 请求，添加 token 字段实现用户独立数据管理与呈现

    # 项目跨域

    用webpack开发配置proxy API代理解决跨域问题，webpack-dev-server，服务端是别人的

    # 导航守卫

    全局守衛router.beforeEach方法，传入参数主要有to，from和next

    首先从store的state中获取到当前token，若已有token即已经登录，所以如果要to到login登录页，直接跳转到home首页，防止重复登录，

    **登录失效：**还可以在beforeEach中发送请求获取用户信息，如果请求失败发现token失效则退出登录直接跳转到登录页

    未登錄：如果是没有获取到token则如果用户想去一些私人页面，比如支付、购物车等页面将跳转到login登录页，且设置/login?redirect=+ path，返回之前准备去的页面

    还有路由内beforeEnter方法，比如在要前往pay支付页前判断是否from来自交易详情页面，否则next(false)无法前往

    以及组件内的beforeRouteEnter方法，在支付成功页面判别是否from支付页面，否则next(false)

    # 面包屑词条增删和点击排序

    在带给服务器的请求中含有搜索参数，

    点击页面商品描述词条后将描述商品的词条字段添加进搜索参数中，然后根据data里的搜索参数通过v-if显示面包屑，

    在每一次增删面包屑/词条时都带着新的搜索参数重新dispatch请求，实时更新搜索到的商品

    排序实现:在搜索参数中同时带有代表排序的order参数，当点击排序按钮（价格、总和）时更改order参数，重新发送请求以新的顺序重新展示商品

    # 非业务逻辑组件封装

    # 分页器

    主要由上一页下一页、...、连续页码若干个、第一页和最后一页组成

    首先通过props传递参数，比如分页器需要知道总页数（商品总数除以每页展示数目pageSize），当前页码，连续页码数（5）

    很重要的自定义事件通信以获得当前所在页码，其次当点击分页器页码按钮时emit自定义事件更改当前页面，在重新发起请求展示数据

    需要computed计算连续页码起始与结束数字--->当前页码加/减连续页码数的一半，并且对于接近第一页（start为1，end为连续页码数）或者最后也一页特殊处理

    掌握start与end数字后通过v-for展示连续页码数5个，并且对于页码数===当前页码数的button赋予active样式。

    而...判别为计算出的start页码大于2与否，end小于总页数-1与否。

    对于上一页和下一页要判断是否到达第一页或者最后一页，给其赋予disabled

    # 轮播图

    而轮播图主要利用swiper插件实现，也是通过props获得页面需要展示的list后v-for展示出来，一个points是如果数据已经修改了但是页面没有跟着渲染则利用$nextTick即可

    # 封装axios请求

    添加token字段，如何实现用户独立数据管理与呈现？

    请求拦截器，在发送请求前做一些事情，interceptors给config的headers请求头添加一个字段，即添加用户独立的token（相当于是对每个接口里相同操作的一个封装）

    还有返回拦截器，接口返回后做的处理，比如对返回的状态进行判断，token过期与否

    # 注册---表单验证

    通过v-validate实现，配合正则表达式，validateAll()时才success

    # mock.js和element-ui

    通过mock.js模拟商城首页的一些商品展示图片，模拟数据以json文件形式传入，同样用axios封装mock请求，只不过在axios.create时更改baseURL

    mock后需要在入口文件引入则可返回请求返回数据

    通过element-ui引入了Button按钮组件以及MessageBox及其alert，比如在支付时弹出支付成功等信息

    引入方式：1. 在入口文件处用Vue.component()注册全局组件或者2.挂载在原型上

    router-view标签展示路由组件

    mapGetters mapState mapSetters

    分别为组件获得getters、state里数据的方法以便更简洁的获取
